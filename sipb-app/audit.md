## 15. Executive Summary

Buat ringkasan maksimal 500 kata yang dapat dikirim langsung kepada pemilik website.

Gunakan bahasa profesional, objektif, dan berikan rekomendasi yang dapat ditindaklanjuti. Jangan hanya menyebutkan masalah, tetapi jelaskan alasan teknis dan dampaknya terhadap pengguna maupun bisnis.

Lakukan audit secara aktif dengan menjelajahi seluruh halaman website yang dapat diakses publik.

Periksa:

- Semua menu navigasi
- Footer
- Header
- Halaman Login
- Dashboard (jika tersedia)
- Contact
- Blog
- FAQ
- Search
- API request
- Console browser
- Network request
- Lighthouse metrics
- Responsive layout
- Error HTTP (4xx/5xx)
- Asset yang gagal dimuat
- Link eksternal dan internal

Jika menemukan bug, sertakan:

- URL halaman
- Langkah reproduksi (Steps to Reproduce)
- Hasil yang diharapkan (Expected Result)
- Hasil aktual (Actual Result)
- Tingkat keparahan (Critical/High/Medium/Low)
- Bukti atau indikasi teknis jika tersedia
- Solusi yang direkomendasikan

Fokus pada temuan yang dapat diverifikasi. Jika ada aspek yang tidak dapat dipastikan hanya dari sisi klien, jelaskan keterbatasan tersebut dan apa yang perlu diperiksa dari sisi server atau kode sumber.
